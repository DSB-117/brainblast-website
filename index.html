Features from the brainblast README

Organized guardrails before the agent starts coding.

brainblast turns requirements into a browsed, cited, structured research run that travels with the project and gives downstream agents fewer ways to hallucinate integrations.

Research workflow

Requirements in, component map out

Auto-detects spec files, inventories APIs, SDKs, auth providers, databases, payment processors, cloud platforms, and chains.

Live source planning

Fetch, don't recall

Plans official docs, registries, changelogs, and rate-limit sources per component, then browses live URLs for every fact.

Questions loop

No dangling uncertainty

Every surfaced question is answered from a live URL or marked unresolvable with a note on where brainblast looked.

Coverage review

Research checks itself

Reviews coverage, re-reads the original requirements, and flags wrong assumptions, missing constraints, and immutable decisions.

Per-component output

Facts, assumptions, inferences, risks

Each component file separates sourced facts, inferred assumptions, derived inferences, rated risks, and resolved questions.

Handoff report

Executive summary and risk heatmap

final-report.md opens with verdict, top risk, irreversible decision, biggest spec gap, and severity counts by component.

Agent instructions

Report pointer auto-injected

Writes an idempotent report pointer into CLAUDE.md or AGENTS.md so the next coding session loads the research.

Machine contract

report.json schema 1.0

Emits stable structured data for components, severity-tagged risks, pre-coding decisions, and requirements corrections.

CI gate

Turn risk into an exit code

--ci mode and a dependency-free gate script can fail builds on CRITICAL or HIGH unresolved risks.

Incremental cache

Cached per name@version

Re-runs reuse unchanged component research from .agent-research/cache and only re-research changed components.

Security model

Docs are data, never instructions

Prompt-injection content found in docs is quoted and flagged, never followed or propagated as an action.

Agent support

Claude Code, OpenClaw, Codex, generic agents

Ships slash commands, adapters, secure tagged installer checksums, update flow, example runs, and validation scripts.

Brainblast Evolved

The fault-prevention layer is becoming an AI training-data platform.

Every time brainblast fix --apply confirms a repair — the original test failing on the broken code and passing on the fix — it manufactures the scarcest artifact in the AI training-data market: a machine-verified, version-pinned, reproducible record of a real security failure and its proven fix. We're building the infrastructure to capture, license, and stream it at scale.

The unit of value

The Verified Trap Instance

A VTI bundles the vulnerable snippet, the corrected snippet, the generated regression test, and the result of re-running both. Not a tag on a commit — an executable claim anyone can reproduce.

Why it's the moat

RED→GREEN is a verifiable reward

Reinforcement learning from verifiable rewards needs ground truth that can be checked programmatically. The model's code passes the test or it doesn't. There is no ambiguity to annotate.

What scrapers can't fake

Freshness and corroboration

Each VTI is pinned to the exact SDK version the failure was found on, and weighted by the number of distinct repositories that confirmed the same fix. Stale, ungraded scraped code can't replicate either.
